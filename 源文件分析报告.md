## 一、项目简介

本工程实现了一套基于 STM32F103C8T6、ESP8266、OneNET 云平台和微信小程序的智能窗帘光温控制系统。系统能够采集环境温度、湿度和光照强度，根据设定阈值自动调节窗帘开合，也支持用户通过微信小程序远程查看数据、切换模式、控制窗帘和调整阈值。

系统核心目标是完成“环境感知、自动联动、远程控制、状态上报、参数下发”的完整闭环。

## 三、系统总体架构

```text
环境传感器
  ├── DHT11：采集温度、湿度
  └── 光敏电阻：采集光照强度

STM32 主控
  ├── 按键输入：本地切换页面、调整阈值、切换模式
  ├── OLED 显示：显示温湿度、光照、阈值、模式和云连接状态
  ├── 步进电机：控制窗帘开合
  └── ESP8266：连接 WiFi 和 OneNET 云平台

OneNET 云平台
  ├── 接收 STM32 上报数据
  ├── 保存设备物模型属性
  └── 转发小程序下发指令

微信小程序
  ├── 首页：显示环境数据和运行状态
  ├── 窗帘页：控制窗帘关闭度
  └── 设置页：调整温度阈值和光照阈值
```

## 四、硬件模块说明

| 模块              | 引脚                  | 功能说明                     |
| ----------------- | --------------------- | ---------------------------- |
| STM32F103C8T6     | 主控芯片              | 系统核心控制器               |
| DHT11             | PA6                   | 采集温度和湿度               |
| 光敏电阻          | PA4 / ADC1_CH4        | 采集环境光照强度             |
| OLED              | PB8 / PB9             | 软件 I2C 显示屏              |
| 28BYJ-48 步进电机 | PA8~PA11              | 通过 ULN2003 驱动窗帘开合    |
| ESP8266           | PA2 / PA3             | USART2 通信，连接 OneNET     |
| LED 指示灯        | PB5~PB7               | 报警或状态提示               |
| 按键              | PC13、PC14、PC15、PA0 | 页面切换、参数调整、模式切换 |

注意：调试串口 USART1 使用 PA9/PA10，而步进电机也占用 PA8~PA11，因此 PA9/PA10 与电机引脚存在复用冲突。答辩演示时如果不需要串口调试，建议以电机功能为主，避免同时使用 USART1 调试输出。

## 五、固件工程说明

当前主要固件工程为 `智能窗帘微改`。该工程使用 Keil 打开 `project.uvprojx` 即可查看源码和编译配置。

### 1. 主程序 `user/main.c`

主程序完成系统初始化和任务调度。初始化内容包括 OLED、按键、光照采集、电机、DHT11、ESP8266 等模块。

主循环采用非阻塞时间片调度，主要周期如下：

| 周期   | 任务                   | 功能                             |
| ------ | ---------------------- | -------------------------------- |
| 100 ms | `Action_Key_Process()` | 按键处理和 OLED 页面刷新         |
| 200 ms | `alarm_int()`          | 自动控制和报警判断               |
| 200 ms | `Wifi_DataUp()`        | 云端下发解析、数据上报、静默重连 |
| 500 ms | `Get_Tem_Hum()`        | 读取温湿度                       |
| 500 ms | `GET_LDR()`            | 读取光照强度                     |

### 2. WiFi 和云平台模块 `API/ESP8266/`

该模块负责 ESP8266 初始化、AT 指令通信、OneNET MQTT 连接、属性上报和属性下发。

核心功能：

| 函数                     | 功能                       |
| ------------------------ | -------------------------- |
| `ESP_01S_Config()`       | 初始化 WiFi 和 MQTT        |
| `AT_Command()`           | 发送 AT 指令并等待应答     |
| `Wifi_ParseRecv()`       | 解析 OneNET 下发的属性设置 |
| `Wifi_DataUp()`          | 周期上报数据并处理下发     |
| `WIFI_ReconnectSilent()` | 离线后静默重连             |
| `Tem_Hum_Publish()`      | 上报温湿度                 |
| `LDR_Publish()`          | 上报光照                   |
| `Heating_curtain()`      | 上报窗帘关闭度             |
| `Heating_mode()`         | 上报自动/手动模式          |
| `Tem_Set_Publish()`      | 上报温度阈值               |
| `Ldr_Set_Publish()`      | 上报光照阈值               |

通信设计特点：

| 设计点       | 说明                                              |
| ------------ | ------------------------------------------------- |
| 下发优先     | 每次 `Wifi_DataUp()` 先解析云端下发，避免下发超时 |
| 立即回复     | 收到 OneNET set 指令后立即发送 `set_reply`        |
| 分步上报     | 每 200 ms 上报一个属性，避免 AT 指令堆积          |
| 离线重连     | 离线后约 10 秒尝试一次静默重连                    |
| 状态严格判断 | 只有 `connect_flag == WIFI_OK` 才认为云端已连接   |

### 3. 按键和自动控制模块 `API/key/`

该模块负责按键扫描、OLED 页面切换、阈值调整、自动控制逻辑。

关键变量：

| 变量        | 默认值 | 范围   | 说明                     |
| ----------- | ------ | ------ | ------------------------ |
| `Auto_Mode` | 1      | 0 或 1 | 1=自动模式，0=手动模式   |
| `Tem_Up`    | 30.0   | -20~60 | 温度阈值                 |
| `ldr_Down`  | 500    | 0~1000 | 光照阈值                 |
| `Ldr_Hyst`  | 100    | 固定值 | 光照迟滞区，防止频繁动作 |

本地按键功能：

| 按键      | 功能                             |
| --------- | -------------------------------- |
| KEY1      | 设置页切换温度阈值和光照阈值     |
| KEY2      | 减小当前选中阈值；手动模式下关窗 |
| KEY3      | 增大当前选中阈值；手动模式下开窗 |
| KEY4 短按 | 切换 OLED 显示页面               |
| KEY4 长按 | 切换自动/手动模式                |

自动控制策略：

| 条件           | 控制逻辑           |
| -------------- | ------------------ |
| 光照很低       | 优先关窗，保护隐私 |
| 温度高且光照强 | 关窗防晒           |
| 温度高且光照弱 | 开窗通风           |
| 温度低且光照强 | 开窗采暖           |
| 温度低且光照弱 | 关窗保温           |

### 4. 电机模块 `API/motor/`

电机模块控制 28BYJ-48 步进电机，实现窗帘开合。

窗帘位置语义已经统一：

| 数值 | 含义     |
| ---- | -------- |
| 0    | 窗帘全开 |
| 100  | 窗帘全关 |

主要函数：

| 函数                  | 功能                   |
| --------------------- | ---------------------- |
| `Motor_Init()`        | 初始化电机 GPIO        |
| `Motor_MoveTo()`      | 移动到目标关闭度       |
| `Motor_Open()`        | 全开窗帘，即移动到 0   |
| `Motor_Close()`       | 全关窗帘，即移动到 100 |
| `Motor_Task()`        | 非阻塞步进电机任务     |
| `Motor_GetPosition()` | 获取当前窗帘关闭度     |

### 5. 传感器模块

| 模块    | 文件                             | 功能                          |
| ------- | -------------------------------- | ----------------------------- |
| DHT11   | `API/DHT11/dht11.c`              | 读取温度和湿度                |
| LDR     | `API/LDR/ldr.c`                  | 读取光敏电阻 ADC 值并换算光照 |
| OLED    | `API/oled/OLED.c`、`OLED_Data.c` | 显示系统状态和中文页面        |
| SysTick | `API/systick/systick.c`          | 提供 1 ms 系统节拍            |
| LED     | `API/ledb/ledb.c`                | 报警指示灯控制                |

OLED 中文显示依赖字模数据，固件中文相关文件建议继续使用 GBK 编码，避免 Keil 和中文字模索引出现异常。

## 六、小程序工程说明

当前小程序工程为 `智能窗帘小程序`，使用微信开发者工具打开该目录即可。

### 1. 全局配置 `app.json`

当前配置包含 3 个页面：

| 页面                    | 功能         |
| ----------------------- | ------------ |
| `pages/index/index`     | 首页数据看板 |
| `pages/curtain/curtain` | 窗帘控制页   |
| `pages/setting/setting` | 参数设置页   |

小程序标题：`智能窗帘光温控制`

已启用按需注入：

```json
"lazyCodeLoading": "requiredComponents"
```

### 2. 首页 `pages/index/`

首页主要展示设备实时状态：

| 显示内容      | OneNET 属性                        |
| ------------- | ---------------------------------- |
| 温度          | `Tmp`                              |
| 湿度          | `Hum`                              |
| 光照          | `ldr`                              |
| 窗帘关闭度    | `curtain`                          |
| 自动/手动模式 | `mode`                             |
| 报警状态      | `Tem_beep`、`Hum_beep`、`CO2_beep` |

首页还支持切换自动模式和手动模式。

### 3. 窗帘页 `pages/curtain/`

窗帘页用于控制窗帘关闭度，支持滑动条和快捷按钮。

设计要点：

| 功能       | 说明                             |
| ---------- | -------------------------------- |
| 关闭度控制 | 0=全开，100=全关                 |
| 自动切手动 | 用户调整窗帘时自动下发 `mode: 0` |
| 操作提示   | 页面提示“切换为手动模式”         |
| 操作记录   | 记录最近控制动作                 |
| 定时刷新   | 周期读取云端状态                 |

### 4. 设置页 `pages/setting/`

设置页用于调整自动控制阈值。

| 参数     | 范围          | OneNET 属性 |
| -------- | ------------- | ----------- |
| 温度阈值 | -20~60 摄氏度 | `tem_set`   |
| 光照阈值 | 0~1000 lx     | `ldr_set`   |

设置页设计了编辑保护逻辑：用户正在拖动或尚未保存时，不直接用云端上报值覆盖当前输入，避免“还没保存就被刷新回去”。

### 5. API 封装 `utils/api.js`

该文件封装 OneNET HTTP API。

| 函数              | 作用             |
| ----------------- | ---------------- |
| `queryProperty()` | 查询设备属性     |
| `setProperty()`   | 下发设备属性     |
| `normalizeMode()` | 统一解析 mode 值 |

测试版和正式版需要在微信公众平台配置合法 request 域名：

```text
https://iot-api.heclouds.com
```

如果只在真机调试能请求成功，而体验版或测试版不能上下发，优先检查微信公众平台后台的“服务器域名 request 合法域名”配置。

## 八、关键问题处理记录

| 问题                         | 当前处理                                                     |
| ---------------------------- | ------------------------------------------------------------ |
| 下发显示失败但实际成功       | 固件收到 set 后立即回复 `set_reply`，前端兼容 OneNET `10411` |
| 自动模式和手动窗帘控制冲突   | 小程序控制窗帘时同步切换为手动模式                           |
| 阈值还没保存就被上报刷新覆盖 | 设置页使用 `editing` 和 `dirty` 状态保护用户输入             |
| 温度阈值范围不一致           | 固件和小程序统一为 -20~60                                    |
| 光照阈值范围不一致           | 固件和小程序统一为 0~1000                                    |
| WiFi/MQTT 失败误判在线       | 初始化失败时 `connect_flag` 置为离线                         |
| 离线后无法恢复               | 增加约 10 秒一次的后台静默重连                               |
| 小程序上传要求按需注入       | `app.json` 已配置 `lazyCodeLoading`                          |
| JSON BOM 导致真机调试失败    | 小程序 JSON 文件已保存为 UTF-8 无 BOM                        |
